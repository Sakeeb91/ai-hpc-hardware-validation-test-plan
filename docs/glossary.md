# Glossary

| Term | Definition |
|---|---|
| Validation | Evidence-based assessment that a system meets intended use and readiness criteria. |
| Verification | Check that a specific requirement or expected state is true. |
| System under test | The exact hardware, firmware, software, and environment being validated. |
| Benchmark | Controlled workload used to measure performance or behavior. |
| Stress test | Workload intended to hold the system under sustained load to expose instability. |
| Soak test | Long-duration stress or workload run used to observe stability over time. |
| Telemetry | Structured measurements such as temperature, power, utilization, clocks, and errors. |
| Pass/fail criteria | Explicit rules used to classify test outcomes. |
| GPU utilization | Percentage of time or resources where the GPU is actively processing work. |
| Memory bandwidth | Rate at which data can be moved to or from memory. |
| Latency | Time required to complete an operation or request. |
| Throughput | Amount of work completed per unit of time. |
| Thermal throttling | Automatic reduction in clocks or performance to control temperature. |
| Power limit | Configured or enforced limit on device power draw. |
| PCIe | Peripheral Component Interconnect Express, the common server expansion bus for GPUs and NICs. |
| NVLink | NVIDIA high-speed GPU interconnect technology. |
| NCCL | NVIDIA Collective Communications Library for multi-GPU communication. |
| CUDA | NVIDIA platform and programming model for GPU acceleration. |
| Driver | Software component that allows the OS and applications to use hardware. |
| Kernel | Core operating system component that manages hardware and system resources. |
| Firmware | Low-level software embedded in hardware devices. |
| BMC | Baseboard Management Controller used for platform management and telemetry. |
| BIOS | Firmware that initializes hardware and exposes platform configuration. |
| Deployment readiness | Assessment that a system is ready, ready with warnings, or not ready for intended use. |
| Issue escalation | Process of routing a documented issue to the correct internal owner, vendor, or OEM. |
| Root-cause analysis | Investigation used to identify the underlying cause of a failure. |
